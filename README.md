# Churn Prediction Intelligence Platform

End-to-end customer/employee attrition prediction system built on the IBM HR Analytics dataset. Covers data science fundamentals through production deployment; designed as a portfolio signal for AI/ML/Data engineering roles.

---

## Tech Stack

| Layer | Tool | Why It Matters in 2026 |
|---|---|---|
| Experiment Tracking | **MLflow** | Industry standard at Databricks, Microsoft, Meta, Adobe |
| Model | **XGBoost + SMOTE** | Go-to gradient boosting; imbalance handling best practice |
| API | **FastAPI** | Async, auto-docs, required skill at 90%+ ML engineer JDs |
| Containerization | **Docker + Compose** | Portable deployment; expected in every ML engineering role |
| Forecasting | **Holt-Winters (statsmodels)** | Explainable time-series for business stakeholders |
| Dashboard | **Plotly Dash** | Python-native; no BI license required |

---

## Dataset

**IBM HR Analytics Employee Attrition**
- Source: [Kaggle](https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset)
- 1,470 employee records; 35 features
- Target: `Attrition` (Yes/No) — ~16% positive rate (class imbalance handled with SMOTE)

## Quickstart (macOS)

### 1. Download dataset

Go to [Kaggle IBM HR dataset](https://www.kaggle.com/datasets/pavansubhasht/ibm-hr-analytics-attrition-dataset), download `WA_Fn-UseC_-HR-Employee-Attrition.csv`, and place it in `data/`.

### 2. Run the full pipeline

```bash
chmod +x run_all.sh
./run_all.sh
```

This will:
- Create a Python virtual environment
- Install all dependencies
- Run EDA and feature engineering
- Train and track 3 models with MLflow
- Forecast 6-month churn by department
- Build and deploy the FastAPI Docker container
- Test the endpoint with sample employees
- Launch the Dash dashboard prompt

### 3. Run steps individually

```bash
source .venv/bin/activate

# EDA + preprocessing
python scripts/01_eda_preprocessing.py

# MLflow training
python scripts/02_mlflow_training.py

# Time-series forecast
python scripts/03_timeseries_forecast.py

# Start API (without Docker)
uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload

# Test endpoint
python scripts/04_test_endpoint.py

# Dashboard
python dashboard/app.py
```

### 4. Docker deployment

```bash
# Install Docker Desktop: https://www.docker.com/products/docker-desktop
cd docker
docker-compose up --build

# Services:
#   API:       http://localhost:8000
#   API docs:  http://localhost:8000/docs
#   MLflow:    http://localhost:5000
#   Dashboard: http://localhost:8050
```

## Feature Engineering

8 new features engineered from the raw IBM dataset:

| Feature | Description | Rationale |
|---|---|---|
| `SalaryAgeRatio` | MonthlyIncome / Age | Detects underpaid younger employees |
| `TenurePerRole` | YearsInCurrentRole / YearsAtCompany | Stagnation signal |
| `SatisfactionComposite` | Average of 4 satisfaction scores | Single engagement index |
| `PromotionLag` | YearsAtCompany - YearsSinceLastPromotion | Career advancement signal |
| `IncomeVsRoleAvg` | Income relative to role average | Equity perception |
| `OvertimeFlag` | Binary OverTime encoding | Burnout indicator |
| `CareerVelocity` | JobLevel / TotalWorkingYears | Progression speed |
| `ManagerTenureRatio` | YearsWithCurrManager / YearsAtCompany | Relationship stability |

## API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/` | Health check |
| `GET` | `/model/info` | Feature list + metadata |
| `POST` | `/predict` | Single employee prediction |
| `POST` | `/predict/batch` | Batch predictions (up to 500) |

**Sample request:**
```python
import requests

employee = {
    "Age": 29, "MonthlyIncome": 2500, "YearsAtCompany": 2,
    "YearsInCurrentRole": 1, "YearsSinceLastPromotion": 2,
    "YearsWithCurrManager": 1, "TotalWorkingYears": 5,
    "JobLevel": 1, "JobSatisfaction": 1, "EnvironmentSatisfaction": 1,
    "RelationshipSatisfaction": 2, "WorkLifeBalance": 1,
    "OverTime": "Yes", "Department": "Sales",
    "JobRole": "Sales Representative", "MaritalStatus": "Single",
    "BusinessTravel": "Travel_Frequently", "EducationField": "Marketing",
    "Gender": "Male", "DistanceFromHome": 25, "Education": 2,
    "NumCompaniesWorked": 4, "PercentSalaryHike": 11,
    "StockOptionLevel": 0, "TrainingTimesLastYear": 1,
    "PerformanceRating": 3, "HourlyRate": 45,
    "DailyRate": 400, "MonthlyRate": 12000, "JobInvolvement": 2,
}

r = requests.post("http://localhost:8000/predict", json=employee)
print(r.json())
# → {"ChurnProbability": 0.82, "ChurnRisk": "High",
#    "PredictedAttrition": true, "TopRiskFactors": [...]}
```
